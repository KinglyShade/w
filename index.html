Main ConfigurationConfiguración Principal

This section explains the main settings in config.yml, which controls the plugin’s general behavior, boss spawning, health bars, abilities, and permissions.

Esta sección explica las configuraciones principales en config.yml, que controla el comportamiento general del plugin, la aparición de jefes, barras de salud, habilidades y permisos.

1. General Settings1. Configuraciones Generales

Basic plugin settings for language, updates, and debugging.

Configuraciones básicas del plugin para idioma, actualizaciones y depuración.

settings:
language: "en_us"
check-for-updates: true
debug-mode: false

Explanation:

  • language: "en_us": Sets the language for plugin messages (e.g., "en_us" for English, "es_es" for Spanish).
  • check-for-updates: true: Notifies admins of new plugin versions.
  • debug-mode: false: When true, logs detailed debug info for troubleshooting.

Explicación:

  • language: "en_us": Establece el idioma para los mensajes del plugin (por ejemplo, "en_us" para inglés, "es_es" para español).
  • check-for-updates: true: Notifica a los administradores sobre nuevas versiones del plugin.
  • debug-mode: false: Cuando es true, registra información detallada de depuración.

2. Logging Settings2. Configuraciones de Registro

Controls logging for debugging purposes.

Controla el registro para fines de depuración.

logging:
enable-logs: true
log-file: "logs/kinglybosses.log"

Explanation:

  • enable-logs: true: Enables logging of plugin events.
  • log-file: "logs/kinglybosses.log": Path to the log file relative to the server directory.
Note: Logging is in beta and may be refined in future updates.

Explicación:

  • enable-logs: true: Habilita el registro de eventos del plugin.
  • log-file: "logs/kinglybosses.log": Ruta al archivo de registro relativa al directorio del servidor.
Nota: El registro está en beta y puede refinarse en futuras actualizaciones.

Controls how bosses spawn globally.

Controla cómo aparecen los jefes globalmente.

general:
spawnRandom: true
spawnInterval: 300
broadcastBossSpawn: true
location:
world: "world"
coordinates:
  x: 0
  y: 0
  z: 0

Explanation:

  • spawnRandom: true: Enables random spawning (currently the only mode, temporarily disabled for other options).
  • spawnInterval: 300: Time between random spawns (300 seconds = 5 minutes).
  • broadcastBossSpawn: true: Announces boss spawns to all players.
  • location: Default spawn point if no specific location is set in boss.yml; uses "world" at coordinates (0, 0, 0).

Explicación:

  • spawnRandom: true: Habilita la aparición aleatoria (actualmente el único modo, deshabilitado temporalmente para otras opciones).
  • spawnInterval: 300: Tiempo entre apariciones aleatorias (300 segundos = 5 minutos).
  • broadcastBossSpawn: true: Anuncia las apariciones de jefes a todos los jugadores.
  • location: Punto de aparición predeterminado si no se establece una ubicación específica en boss.yml; usa "world" en las coordenadas (0, 0, 0).

Customizes the boss health bar display.

Personaliza la visualización de la barra de salud del jefe.

bossHealthBar:
active: true
title: "&c&lBoss: &7{boss}"
color: "RED"
style: "SOLID"
progress: 1.0
update_interval: 0.1
display_distance:
enabled: true
radius: 100
permission_based_display:
enabled: false
permission: "bossbar.view"
hide_on_death:
enabled: true

Explanation:

  • active: true: Enables the health bar.
  • title: Displayed text; {boss} is replaced by the boss’s name.
  • color: "RED": Bar color (options: red, blue, green, etc.).
  • style: "SOLID": Bar style (e.g., solid, segmented_6).
  • progress: 1.0: Initial health percentage (0 to 1).
  • update_interval: 0.1: Updates every 0.1 seconds.
  • display_distance: Limits visibility to players within 100 blocks.
  • permission_based_display: When enabled, restricts visibility to players with the "bossbar.view" permission.
  • hide_on_death: true: Hides the bar when the boss dies.

Explicación:

  • active: true: Habilita la barra de salud.
  • title: Texto mostrado; {boss} se reemplaza por el nombre del jefe.
  • color: "RED": Color de la barra (opciones: red, blue, green, etc.).
  • style: "SOLID": Estilo de la barra (por ejemplo, solid, segmented_6).
  • progress: 1.0: Porcentaje inicial de salud (0 a 1).
  • update_interval: 0.1: Se actualiza cada 0.1 segundos.
  • display_distance: Limita la visibilidad a jugadores dentro de 100 bloques.
  • permission_based_display: Cuando está habilitado, restringe la visibilidad a jugadores con el permiso "bossbar.view".
  • hide_on_death: true: Oculta la barra cuando el jefe muere.

  • phases: Cambia el comportamiento del jefe en umbrales de salud.

Complete Boss CreationCreación Completa de Jefes

This section details how to create bosses using the boss.yml configuration, with examples based on three unique bosses: Epic Boss, Mighty Guardian, and Shadow Beast.

Esta sección detalla cómo crear jefes usando la configuración boss.yml, con ejemplos basados en tres jefes únicos: Epic Boss, Mighty Guardian y Shadow Beast.

1. Basic Structure1. Estructura Básica

Every boss starts with a unique identifier under the 'bosses' section.

Cada jefe comienza con un identificador único bajo la sección 'bosses'.

bosses:
  'epic_boss':
    # Configuration goes here

Explanation: The bosses key is the root of all boss configurations. Each boss is defined with a unique name (e.g., 'epic_boss') that acts as its identifier in the file. This name doesn’t affect gameplay but must be unique to avoid conflicts.

Explicación: La clave bosses es la raíz de todas las configuraciones de jefes. Cada jefe se define con un nombre único (por ejemplo, 'epic_boss') que actúa como su identificador en el archivo. Este nombre no afecta el juego, pero debe ser único para evitar conflictos.

2. Core Settings2. Configuraciones Esenciales

Define the boss's identity, type, and basic stats.

Define la identidad, tipo y estadísticas básicas del jefe.

'epic_boss':
  boss-id: "3"
  level: 10
  enabled: true
  mob-type: ENDER_DRAGON
  display-name: 'Dragón del Caos'
  health: 1000
  attack-damage: 25
  movement-speed: 0.3

Explanation: These are the essential settings for any boss:

  • boss-id: "3": A unique identifier for the boss, separate from its name.
  • level: 10: Indicates difficulty; higher levels may increase rewards or stats.
  • enabled: true: Must be true for the boss to be active in the game.
  • mob-type: ENDER_DRAGON: Defines the entity type (use exact Minecraft entity names).
  • display-name: 'Dragón del Caos': The visible name players see, supports color codes.
  • health: 1000: Total health points; higher values make the boss tougher.
  • attack-damage: 25: Damage per hit; balance this with health for difficulty.
  • movement-speed: 0.3: Speed of movement; 0.3 is moderate, adjust for balance.

Explicación: Estas son las configuraciones esenciales para cualquier jefe:

  • boss-id: "3": Un identificador único para el jefe, separado de su nombre.
  • level: 10: Indica la dificultad; niveles más altos pueden aumentar recompensas o estadísticas.
  • enabled: true: Debe ser true para que el jefe esté activo en el juego.
  • mob-type: ENDER_DRAGON: Define el tipo de entidad (usa nombres exactos de entidades de Minecraft).
  • display-name: 'Dragón del Caos': El nombre visible para los jugadores, soporta códigos de color.
  • health: 1000: Puntos totales de vida; valores más altos hacen al jefe más resistente.
  • attack-damage: 25: Daño por golpe; equilibra esto con la vida para la dificultad.
  • movement-speed: 0.3: Velocidad de movimiento; 0.3 es moderada, ajústala para equilibrar.

Control how and when the boss spawns or respawns.

Controla cómo y cuándo aparece o reaparece el jefe.

# Epic Boss Spawn Settings
'epic_boss':
  chance: 30
  spawn-message: '¡Un dragón épico ha emergido de las profundidades!'
  spawn-sound: 'ENTITY_ENDER_DRAGON_AMBIENT'
  spawnZone: true
  activespawnInterval: true
  spawnInterval: 1200  # 20 minutes
  allowBossRespawn: true
  MaxRespawn: 3
  respawnInterval: 3600  # 1 hour
  RespawnMessages:
    - "¡El temible {bossname} ha regresado para causar estragos!"

Explanation:

  • chance: 30: 30% probability of spawning naturally.
  • spawn-message: Displays this message to players when the boss spawns.
  • spawn-sound: Plays this sound (Minecraft sound ID) on spawn.
  • spawnZone: true: Uses a custom spawn area if defined.
  • activespawnInterval: true: Enables a custom spawn interval.
  • spawnInterval: 1200: Spawns every 20 minutes (in seconds).
  • allowBossRespawn: true: Allows the boss to respawn after defeat.
  • MaxRespawn: 3: Limits respawns to 3 times.
  • respawnInterval: 3600: Respawns after 1 hour (in seconds).
  • RespawnMessages: List of messages shown on respawn; {bossname} is replaced by the display name.

Explicación:

  • chance: 30: 30% de probabilidad de aparecer naturalmente.
  • spawn-message: Muestra este mensaje a los jugadores cuando aparece el jefe.
  • spawn-sound: Reproduce este sonido (ID de sonido de Minecraft) al aparecer.
  • spawnZone: true: Usa un área de aparición personalizada si está definida.
  • activespawnInterval: true: Habilita un intervalo de aparición personalizado.
  • spawnInterval: 1200: Aparece cada 20 minutos (en segundos).
  • allowBossRespawn: true: Permite que el jefe reaparezca tras ser derrotado.
  • MaxRespawn: 3: Limita las reapariciones a 3 veces.
  • respawnInterval: 3600: Reaparece después de 1 hora (en segundos).
  • RespawnMessages: Lista de mensajes mostrados al reaparecer; {bossname} se reemplaza por el nombre visible.

Personaliza ataques, desencadenantes y resistencias.

  • triggers: Events triggered on specific conditions; on_defeat runs the "spawn_epic_reward" action when the boss dies.

Explicación:

  • fire_damage: false: El jefe es inmune al daño por fuego.
  • attack: true: Habilita habilidades de ataque personalizadas.
  • attacks: Lista de IDs de ataques definidos en otro lugar (por ejemplo, en attacks.yml).
  • triggers: Eventos que se activan en condiciones específicas; on_defeat ejecuta la acción "spawn_epic_reward" al morir el jefe.

Set rewards for top performers.

Establece recompensas para los mejores jugadores.

# Epic Boss Rewards
'epic_boss':
  rewards:
    top_1:
      reward:
        type: command
        commands:
          - 'give %player% elytra 1'
          - 'give %player% dragon_egg 1'
    top_2:
      reward:
        type: item
        item: 'dragon_scale'
        amount: 5
    top_3:
      reward:
        type: itemgroup
        item: 'epic_loot'
        amount: 3

Explanation:

  • rewards: Defines rewards for the top 3 damage dealers.
  • top_1: For the highest damage dealer; type: command runs server commands (e.g., giving elytra and a dragon egg).
  • top_2: type: item gives a specific item (dragon_scale) with an amount.
  • top_3: type: itemgroup gives items from a predefined group (epic_loot).

Explicación:

  • rewards: Define recompensas para los 3 mejores infligidores de daño.
  • top_1: Para el mayor infligidor de daño; type: command ejecuta comandos del servidor (por ejemplo, dar élitros y un huevo de dragón).
  • top_2: type: item da un objeto específico (dragon_scale) con una amount.
  • top_3: type: itemgroup da objetos de un grupo predefinido (epic_loot).
